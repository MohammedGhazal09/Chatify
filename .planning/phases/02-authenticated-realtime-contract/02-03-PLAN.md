<threat_model>
Trust boundaries:
- Presence state crosses from backend authenticated socket maps to other users' clients.
- Reconnect crosses from transient socket delivery into durable HTTP/query state.
- Multi-socket disconnect timers cross from asynchronous runtime state into persisted `User.isOnline` and `lastSeen`.

STRIDE register:
- Information disclosure: Hidden presence could leak to users sharing a room. Mitigation: check `showOnlineStatus` server-side before every broadcast and target only authorized contacts.
- Tampering: Reconnect could trust stale client cache. Mitigation: invalidate/refetch chats, selected messages, unread counts, and presence from server truth after `socket:ready`.
- Denial of service: Debounce timers can leak across disconnect/reconnect tests. Mitigation: clear timers when a user reconnects and during test teardown.
- Repudiation: Presence transitions need deterministic test evidence. Mitigation: real socket tests cover first connect, second socket connect, one disconnect, final disconnect, privacy disabled, and reconnect readiness.
</threat_model>
