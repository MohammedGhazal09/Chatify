# Phase 38 Plan 03 - Frontend Spaces Workspace And Channel UI

## Objective

Add the spaces workspace to the existing chat shell and route selected channels through the existing conversation pane.

## Scope

- Frontend space/channel types and API client methods.
- Query/mutation hooks for space list, create space, member changes, channel list, and create channel.
- Sidebar Conversations/Spaces switch and space/channel list UI.
- Create space and create channel dialogs with username-based member input.
- Channel selection that feeds existing `ConversationPane`, `useMessages`, and socket behavior.
- Component/hook tests for empty, loading, create, select, access-denied, and long-name states.

## Acceptance Criteria

- [x] User can switch to Spaces mode and see only authorized spaces.
- [x] User can create a space and select its default channel.
- [x] Owner/admin controls render only for allowed roles.
- [x] Channel selection uses the existing timeline/composer without duplicating message logic.
- [x] Email is absent from member/invite/channel UI.
