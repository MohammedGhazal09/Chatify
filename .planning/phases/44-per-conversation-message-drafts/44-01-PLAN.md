# Phase 44 Plan 01 - Local Conversation Draft Lifecycle

## Objective

Add frontend-local per-conversation text drafts to the existing chat composer and sidebar without changing backend APIs or message persistence.

## Scope

- Add a focused conversation draft hook and storage helpers under `Frontend/Chatify/src/pages/chat/hooks`.
- Wire the hook to `ChatPage` `messageInput`, selected chat id, accessible chat ids, send success cleanup, and private-state cleanup.
- Add bounded draft indicators to `ChatSidebar` and `ChatListItem`.
- Keep encrypted conversation row previews generic while preserving composer restoration locally.
- Add focused hook and sidebar tests.

## Threat Model

- **Threat:** Draft text leaks across accounts on a shared browser.
  **Mitigation:** Storage key includes authenticated user id; logout/session cleanup removes the key.
- **Threat:** Draft text leaves the device.
  **Mitigation:** No HTTP, Socket.IO, backend model, or log writes are added for drafts.
- **Threat:** Encrypted draft plaintext is exposed in conversation list/search.
  **Mitigation:** Encrypted rows render generic draft copy and sidebar search ignores encrypted draft text.
- **Threat:** Malformed localStorage data breaks the chat page.
  **Mitigation:** Storage parsing falls back to an empty map and keeps in-memory typing.
