# Plan 19-04: Empty Offline Blocked And Failure State Polish

## Objective

Consolidate the messenger's edge-state language and accessibility so first-run, no-chat, no-results, offline, reconnecting, blocked, unavailable-call, failed-upload, and failed-send states feel consistent on desktop and mobile.

## Tasks

1. Read `19-CONTEXT.md`, `19-RESEARCH.md`, `ChatStateView.tsx`, sidebar, conversation pane, message list, message search results, composer, header, more menu, and related tests.
2. Extend `ChatStateView` only where it improves shared state behavior, such as tone, icon slot, action layout, live-region intent, or mobile spacing.
3. Polish sidebar empty/loading/error/no-results states with useful next actions and accessible copy.
4. Polish conversation empty, message-loading error, no message search result, below-minimum search, offline, reconnecting, blocked, and session-expired states with consistent copy and recoverable actions.
5. Polish failed-send and failed-upload states so retry/dismiss/cancel controls remain keyboard-accessible, stable, and clear.
6. Polish unavailable-call copy in header and menu states so disabled call controls explain direct-chat, online, socket, browser support, permission, or blocked-user reasons consistently.
7. Check mobile constraints for long copy, buttons, banners, composer dock, and sidebar drawer so text does not overlap or overflow.
8. Add focused tests for state copy, live regions, keyboard-operable actions, mobile-relevant structure, and no duplicate or hidden controls.

## Success Criteria

All major messenger edge states provide useful direction, accessible controls, stable layout, and consistent language without broad rewrites.

## Must Haves

- Do not replace working message, unread, delivery, blocking, or call logic while polishing UI copy.
- Every interactive recovery action must be reachable by accessible role/name.
- Mobile copy must fit without horizontal overflow or overlapping the composer/header.

## Artifacts This Phase Produces

- Polished shared state components
- Updated component tests for state surfaces
- CSS adjustments only where needed for responsive stability
