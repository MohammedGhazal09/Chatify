# Phase 42 Plan 02 - Frontend Trusted Conversation Onboarding

## Objective

Expose contact request send/list/accept/decline/cancel states inside the existing messenger UI.

## Scope

- Add contact request types and API methods.
- Add TanStack Query hooks for request list and lifecycle mutations.
- Update `NewChatDialog` copy and submit handling for request sent vs existing chat returned.
- Add a compact contact request section to the start-conversation/sidebar flow.
- Keep group creation, existing direct continuation, and encrypted existing-chat behavior intact.
- Add focused component/hook tests.

## Acceptance Criteria

- [x] Direct new-chat copy says a request is sent before a new private chat opens.
- [x] Sending a request keeps the user in a clear pending state instead of pretending a chat opened.
- [x] Incoming requests expose accept and decline controls.
- [x] Outgoing requests expose pending and cancel controls.
- [x] Accepting a request merges/selects the returned chat.
- [x] Component tests cover empty/loading/error/pending/action states.
