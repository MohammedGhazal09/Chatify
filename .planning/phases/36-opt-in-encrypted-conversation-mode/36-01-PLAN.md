# Phase 36 Plan 01 - Backend Encrypted Conversation And Message Contract

## Objective

Add the backend data model and request contracts for opt-in encrypted conversations and encrypted message envelopes while preserving standard chat behavior.

## Scope

- `Chats.encryptionMode` model field and serialization.
- Direct-key behavior for separate standard and encrypted direct conversations.
- Create direct/group chat payload normalization for `standard` and `e2ee_v1`.
- `Messages.messageType = encrypted`, encrypted payload schema, and safe serialization.
- Encrypted message send validation that rejects plaintext text/attachments in `e2ee_v1` chats.
- Backend tests for standard compatibility, encrypted chat creation, encrypted send, plaintext rejection, and idempotency.

## Acceptance Criteria

- [ ] Existing standard direct/group creation tests pass unchanged.
- [ ] A standard and encrypted direct conversation can coexist for the same two members.
- [ ] Encrypted messages store an encrypted envelope and empty plaintext `text`.
- [ ] Plaintext sends and normal attachment upload are rejected in encrypted conversations.
- [ ] Idempotent encrypted sends compare encrypted envelope fingerprints, not plaintext text.
