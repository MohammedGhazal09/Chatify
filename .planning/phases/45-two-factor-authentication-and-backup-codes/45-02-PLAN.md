# Phase 45 Plan 45-02: Frontend 2FA Login And Settings

## Goal

Wire the frontend to the backend 2FA contract with a hardened login challenge and Settings management UI.

## Scope

- Add typed auth API methods for 2FA.
- Update login mutation behavior so challenge responses do not fetch logged-in user prematurely.
- Add a login second-factor step with invalid/expired/loading states.
- Add Settings 2FA status, setup, confirm, backup-code display, regenerate, and disable controls.
- Add focused frontend tests for login challenge and Settings 2FA management.
- Run visual QA across desktop and mobile.

## Acceptance

- Existing non-2FA login still redirects after fetching the user.
- 2FA login shows the challenge step and completes after verification.
- Settings exposes setup/enable/disable/regenerate without layout overflow or hidden error states.
- Backup codes render only after enable/regenerate and clear when the modal closes.
- Frontend tests, lint, build, visual QA, and review pass.
