# Plan 18-03: Quality Scripts And Operational Runbooks

## Objective

Make operational checks and recovery procedures repeatable from PowerShell without requiring maintainers to reconstruct commands from phase artifacts.

## Tasks

1. Read `18-CONTEXT.md`, root/backend/frontend `package.json`, Phase 14 and Phase 15 acceptance artifacts, `.env.example` files, and existing README files.
2. Replace the root placeholder `test` script with a non-destructive local gate and add root scripts for backend tests, frontend tests, frontend lint, frontend build, full quality, local e2e, production smoke, and ops guard checks.
3. Add any package-local helper scripts needed to make root scripts clear and reusable.
4. Create `docs/operations/` runbooks for local startup, deployment verification, production smoke setup, incident triage, rollback, and credential rotation. Use exact commands and placeholder env values only.
5. Update README with a short operations section that points to the runbooks and root scripts.

## Verification

- `npm run quality` exists at the root and runs backend tests, frontend tests, lint, and build through npm `--prefix` calls.
- `npm run smoke:prod` exists at the root and delegates to the existing frontend production Playwright command.
- Runbooks include prerequisites, procedure, verification, troubleshooting, rollback or escalation where relevant.
- No runbook contains real smoke account values, tokens, cookies, reset codes, database URLs, TURN credentials, or API keys.

## Success Criteria

An operator can start, test, smoke, triage, roll back, and rotate credentials using versioned docs and commands.

## Must Haves

- Scripts must be PowerShell-friendly and avoid POSIX-only command chains.
- Runbooks must preserve Phase 14/15/17 blocked status unless configured smoke evidence exists.
