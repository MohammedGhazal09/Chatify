<threat_model>
Trust boundaries:
- Browser to Socket.IO handshake crosses from untrusted client input into trusted backend session state.
- `accessToken` cookie parsing crosses from raw headers into JWT verification.
- Socket maps cross from connection-level state into user-specific delivery and presence decisions.

STRIDE register:
- Spoofing: A client can emit another user's id through `user:connect`. Mitigation: ignore the payload and derive identity only from a verified cookie.
- Tampering: A client can send malformed or unsigned cookies. Mitigation: verify JWT signature and expiration with the shared helper before connection.
- Information disclosure: Auth failures could expose token details. Mitigation: stable generic error codes and no raw token/cookie logging.
- Denial of service: Test servers or sockets can leave open handles. Mitigation: helper-owned teardown closes client sockets, Socket.IO, and HTTP server.
</threat_model>
