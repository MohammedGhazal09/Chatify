# 48-02 Plan: Frontend Saved Messages Workflow

## Goal

Expose saved-message actions, state, and a global saved-message dialog in the chat UI.

## Requirements

- V2-SAVED-01
- V2-SAVED-03
- V2-SAVED-04
- V2-SAVED-05
- V2-SAVED-06

## Tasks

1. Extend frontend message contracts:
   - add optional `savedByRequester` and `savedAt` fields to `Message`
   - add `SavedMessage` type for saved-list rows
   - add saved-message response interfaces in `messageApi.ts`
2. Add message API methods:
   - `listSavedMessages`
   - `saveMessage`
   - `unsaveMessage`
3. Add React Query saved-message hooks:
   - `savedMessagesQueryKey`
   - `useSavedMessages`
   - `useSaveMessage`
   - `useUnsaveMessage`
   - cache update helper that patches message `savedByRequester`
4. Extend `MessageActionMenu`:
   - add `Save message` / `Unsave message`
   - keep encrypted messages eligible but deleted-for-everyone messages ineligible
   - preserve disabled active-action behavior
5. Extend `MessageBubble`:
   - show a compact saved bookmark indicator in metadata when `savedByRequester` is true
   - avoid wrapping over timestamp or delivery status on mobile
6. Add `SavedMessagesDialog`:
   - loading, empty, error, populated states
   - safe preview generation for text, attachments, voice/media/file, and encrypted messages
   - jump and unsave row actions
   - Escape/close focus behavior
7. Wire the chat route:
   - sidebar footer opens saved dialog
   - save/unsave action handlers show toasts
   - saved dialog jump selects target chat/channel and reuses context/highlight behavior
8. Add frontend tests:
   - `SavedMessagesDialog.test.tsx`
   - extend `MessageActionMenu.test.tsx`
   - extend message cache/query tests where saved state is patched
9. Add Phase48 visual QA runner and capture desktop/mobile saved workflows.

## Risks

- The saved dialog can become too wide or text-heavy on mobile if long group names or filenames are not clamped.
- Cross-chat jump can fail if the target chat is a space channel and the workspace state is not selected correctly.
- Cache patches can accidentally drop unrelated message fields if they replace messages instead of merging them.

## Threat Model

- **Private metadata leakage:** Saved-list UI could display sender email or hidden content. Mitigation: use public identity fields and generic encrypted/deleted fallbacks only.
- **UI confusion:** A saved indicator could be mistaken for shared pin state. Mitigation: copy uses "Saved" and pin copy remains "Pinned"; icons are compact and requester-specific.
- **Unauthorized restore:** Jumping from saved list could reveal hidden messages. Mitigation: jump uses existing context API, which enforces visibility again.
