# Plan 24-02: Group-Originated Call Signaling

## Objective

Allow authorized group chats to start voice/video call sessions that ring reachable group members and connect one accepting member through the existing WebRTC path.

## Must-Haves

- Cover `24-SPEC.md` requirements 3, 4, and 5.
- Cover `24-CONTEXT.md` decisions D-07 through D-19.
- Preserve direct call payload compatibility.
- Server derives group recipients from authenticated chat membership; clients do not send trusted participant ids.
- No production or TURN readiness claim is made from local tests.

## Tasks

1. Extend call session persistence and serialization with group metadata while keeping direct `calleeId` compatibility.
2. Add server helpers to derive group call recipients and active participant ids from `chat.members`, `callerId`, `deliveredTo`, and the first accepted member.
3. Update `call:start` to accept group chats, ring reachable non-caller group members, fail with `callee_unavailable` when nobody is reachable, and reject outsiders through existing membership checks.
4. Update accept/reject/end/sync/signaling authorization for group-originated sessions while preserving stale-call and busy-user protection.
5. Forward offer/answer/ICE only between the caller and accepted group peer for this phase.
6. Update frontend call session types and call controller peer selection so incoming and accepted group sessions use the correct member as the visible peer.
7. Replace the existing backend group-rejection test with group start, incoming delivery, outsider rejection, unreachable failure, and first-acceptance coverage.

## Verification

- `cd Backend/Chatify; npm test -- --run test/socket/socket.calls.test.mjs`
- `cd Frontend/Chatify; npm test -- useCallController.test.tsx`
- `cd Frontend/Chatify; npm run lint`
- `cd Frontend/Chatify; npm run build`

## Success Criteria

- Authorized group calls start and ring reachable group members.
- Unauthorized group call starts and unreachable groups fail with structured acks.
- One group member can accept and become the active media peer.
- Direct call tests still pass.
- No user-facing call copy claims full conference-room support.
