# Plan 20-02: Signup Username Field And Existing-User Mandatory Setup Gate

## Objective

Add the frontend username experience: signup username validation/input, typed username setup API/mutation, a mandatory setup page for existing authenticated users, and route gating that cannot be bypassed by refresh.

## Tasks

1. Read `20-CONTEXT.md`, `20-UI-SPEC.md`, `signup.tsx`, `login.tsx`, `App.tsx`, `protectedRoute.tsx`, auth store/types/API/hooks, and existing signup/auth route tests.
2. Add mirrored frontend username validation in `utils/usernameValidation.ts` and wire it into `validationSchemas.tsx` for signup and setup forms.
3. Update auth/user types and Zustand auth state so authenticated users may include `username`, while existing users without username are still representable.
4. Update `authApi.signup` and `SignupData` to include username; add the username field to `signup.tsx` between the name row and email field using the UI-SPEC copy, helper text, error association, and stable loading/error states.
5. Add `userApi.setUsername` and a `useSetUsername` mutation that calls the CSRF-protected endpoint, updates auth state on success, and maps validation/duplicate errors to the username field where possible.
6. Add `SetupUsername.tsx` with the UI-SPEC layout, copy, one username field, root error region, stable submit button, and no skip/later path.
7. Extend `ProtectedRoute` or add a sibling guard so authenticated users missing username are redirected to `/setup-username` from `/` and future protected feature routes, while `/setup-username` remains reachable only to authenticated users missing username.
8. Add frontend tests for signup username validation/rendering/submission, setup mutation behavior, route refresh gating, users with username bypassing setup, and users without username reaching setup but not chat.

## Verification

- `cd Frontend/Chatify; npm test -- --run src/utils/validationSchemas.test.ts src/pages/signup/signup.test.tsx src/hooks/useAuthQuery.test.tsx src/components/protectedRoute.test.tsx`
- `cd Frontend/Chatify; npm run build`

## Success Criteria

New users must provide a username during signup, and existing authenticated users without username cannot reach chat until setup succeeds.
