## Decisions Made

- The local smoke uses `127.0.0.1` for both frontend and backend URLs to avoid localhost/127 cookie site mismatch.
- The configured local backend port `3000` was not Chatify in this environment, so the verified local run used an isolated backend on port `5000`.
- The first isolated local backend attempt against the configured database was blocked by MongoDB connectivity; using `mongodb-memory-server` allowed real HTTP, Socket.IO, cookie, and persistence paths without relying on external database access.

## Deviations from Plan

- Local browser proof used an in-memory MongoDB backend instead of the developer's configured local MongoDB because the configured database timed out during smoke account provisioning.
- Production smoke was not run live because required opt-in credentials and deploy identifiers were not available. This is recorded as blocked, not passed.
- Roadmap progress remains at `2/3`; Phase 10.1 is not marked complete because production evidence is still blocked.

## Issues Encountered

- Running two Playwright greps in parallel caused Vite port `4177` collisions. Subsequent smoke commands were run serially.
- A backend full-test run hit a transient `mongodb-memory-server` startup timeout; the immediate rerun passed all backend tests.
- Browser login initially failed with `localhost:5000` API URLs because the frontend origin was `127.0.0.1:4177`; switching API/socket URLs to `127.0.0.1:5000` fixed cookie behavior.
