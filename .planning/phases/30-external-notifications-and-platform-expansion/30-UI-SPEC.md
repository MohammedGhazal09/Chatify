# Phase 30 UI Specification

No runtime UI changes are implemented in this phase.

## Future Notification UI

- Settings must separate local browser alerts from server push and email notifications.
- Push/email toggles must be off by default.
- Email notification settings must include unsubscribe and quiet-hours controls.
- Encrypted conversation notification copy must be generic and visibly non-previewable.
- Muted conversations must clearly suppress external notifications.

## Future Platform UI

- Private spaces should not be a landing-page-style feature; they should behave like a focused workspace/conversation tool.
- Bot/integration installs must show requested scopes before approval.
- Revocation must be one action from the integration detail surface.

## Recommendation

Recommendation: do not show channels, spaces, bots, or integrations in navigation until the backend permission model exists.
