# Phase 34 Plan 02 - Frontend Filters And Jump-To-Result

## Objective

Expose advanced message search filters in the existing conversation pane and make unloaded results jumpable through the backend context endpoint.

## Scope

- Message search types and API params.
- `useMessageSearch` filter query key and context fetch helper.
- Search filter controls in the conversation pane.
- Result metadata rendering.
- Cache merge, scroll, and highlight behavior for unloaded results.
- Frontend tests for filters and jump behavior.

## Acceptance Criteria

- [ ] Search filters can be changed without losing selected conversation state.
- [ ] TanStack Query keys include normalized filters.
- [ ] Loaded results still jump immediately.
- [ ] Unloaded results fetch a context window, merge cache, scroll, and highlight.
- [ ] Clearing search resets query and filters.
- [ ] Tests cover filter API params, rendering, and jump fallback.
