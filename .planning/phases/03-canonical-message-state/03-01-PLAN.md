<threat_model>
Trust boundaries:
- Browser request bodies cross into authenticated backend message creation and mutation.
- Socket event payloads cross into persisted message status updates.
- Message visibility filters cross from private persisted records into user-specific history, unread counts, and realtime events.

STRIDE register:
- Spoofing: A client can submit `sender` or forged chat/message ids. Mitigation: derive sender from `req.userId` and check membership through existing access helpers.
- Tampering: A retry can reuse `clientMessageId` with changed content. Mitigation: compare normalized content and return `409` instead of overwriting or silently accepting.
- Information disclosure: Unauthorized message ids can reveal private resources. Mitigation: generic private-resource failures and no leaked message text in tombstones.
- Repudiation: Duplicate HTTP retry could emit multiple events. Mitigation: idempotent branch returns existing payload and records no duplicate side effects.
- Denial of service: Unbounded reactions or invalid payloads can bloat documents. Mitigation: max 32-character emoji strings and max 50 reactions per message.
- Elevation of privilege: Sender-only delete/edit could be bypassed. Mitigation: route-level ownership checks plus regression tests.
</threat_model>
