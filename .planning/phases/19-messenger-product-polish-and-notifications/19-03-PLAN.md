# Plan 19-03: Account Session And Multi-Tab Edge-State Polish

## Objective

Make logout, refresh failure, expired session, account state, and multi-tab auth changes feel intentional and recoverable while clearing private chat data promptly.

## Tasks

1. Read `19-CONTEXT.md`, `axios.ts`, `useAuthQuery.ts`, `authstore.ts`, protected/public routes, Settings modal, sidebar, `ConversationPane.tsx`, and current auth/session tests.
2. Add a small session broadcast utility or hook that emits and listens for logout/auth-expired events through `BroadcastChannel` with a storage-event fallback.
3. Ensure logout and refresh-failure paths clear auth state, presence state, selected private chat state where appropriate, and TanStack Query private caches across active tabs.
4. Preserve the existing `chatify:auth-expired` event behavior and make it compatible with the multi-tab session channel.
5. Polish account/session copy in Settings, sidebar, and session-expired chat state so users understand whether they can retry, sign in, or continue safely.
6. Verify private conversation content is hidden before or at the same time as navigation to login on auth expiry.
7. Add tests for current-tab logout, multi-tab logout, refresh-failure propagation, session-expired recovery UI, and no cross-user selected-chat or notification preference leakage.

## Verification

- `cd Frontend/Chatify; npm test -- --run src/api/axios.test.ts src/hooks/useAuthQuery.test.tsx src/hooks/useSessionBroadcast.test.tsx`
- `cd Frontend/Chatify; npm test -- --run src/pages/chat/components/ConversationPane.test.tsx src/components/SettingsModal.test.tsx`
- Existing Playwright auth-expired smoke still passes.

## Success Criteria

Session edge cases clear private UI state and recover cleanly in one tab or multiple tabs without confusing stale chat content.

## Must Haves

- Auth-expired and logout flows must not leave private message text visible after the user is no longer authenticated.
- Broadcast payloads must not include emails, tokens, cookies, or message data.
- Multi-tab behavior must degrade safely when `BroadcastChannel` is unavailable.

## Artifacts This Phase Produces

- Session broadcast utility or hook
- Auth/session polish tests
- Updated session-state UI copy
