# Phase 43 Plan 02 - Frontend Quoted Reply Interaction

## Objective

Wire the existing reply UI into the durable message contract, then render quote previews in composer and message bubbles with hardened mobile, keyboard, deleted, unavailable, and encrypted states.

## Scope

- Extend frontend chat types and message API payload handling with `replyToMessageId` and `replyTo`.
- Preserve reply metadata in optimistic sends, retry/failed state, cache merge, socket updates, and chat latest-message cache.
- Update composer preview copy/fallbacks and disable encrypted durable replies with honest copy.
- Add message bubble quote rendering and source jump behavior using existing local scroll/highlight and message-context loading.
- Add focused component/hook tests for composer, bubble, send mutation, cache merge, and quote jump behavior.

## Acceptance Criteria

- [x] Selecting Reply shows a hardened composer preview with sender, preview/fallback, cancel, and keyboard support.
- [x] Sending a standard reply includes `replyToMessageId`, creates an optimistic `replyTo`, and clears reply state on success.
- [x] Failed optimistic replies retain quote context for retry clarity.
- [x] Message bubbles render quote blocks without overflow on long/RTL/emoji text.
- [x] Quote click jumps/highlights loaded sources or loads context before highlighting.
- [x] Encrypted conversations show a clear limitation and do not send decrypted quote preview metadata.
