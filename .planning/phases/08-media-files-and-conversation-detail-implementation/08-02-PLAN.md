<threat_model>
Primary assets: selected browser `File` objects, local object URLs, protected preview/download endpoints, chat detail query data, optimistic attachment summaries, and truthful security copy.
Threats: static demo media replacing live state, local object URLs leaking after unmount or retry, preview/download actions bypassing protected routes, long filenames or failed states breaking mobile layout, unsupported controls implying capabilities that do not exist, and non-living visual constraints being violated by placeholders or fixtures.
Mitigations: keep transport in `messageApi`, cache ownership in TanStack Query hooks and `messageCache`, revoke object URLs on removal/send/failure/unmount, build preview/download actions from protected backend route helpers, render loading/empty/error/unauthorized states from query state, disable or remove unsupported controls honestly, constrain text/media layout across mobile and desktop, and use abstract non-living placeholders only.
Residual risk: browser-local selected files cannot survive reload; the UI must show reattach-required behavior instead of pretending retry can continue.
</threat_model>
