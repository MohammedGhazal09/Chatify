<threat_model>
Trust boundaries:
- Client event payloads cross from untrusted chat/message ids into database-backed membership checks.
- Room membership controls which sockets receive chat and presence data.
- Controller-origin private emits cross from persisted HTTP mutations into user-specific Socket.IO delivery.

STRIDE register:
- Spoofing: A valid user can claim another chat id. Mitigation: check `Chats.members` using `socket.data.userId` for every event.
- Tampering: A client can submit a `messageId` with a forged `chatId`. Mitigation: load the persisted message and derive `chatId` server-side.
- Information disclosure: Specific 404/403 distinctions can reveal private resources. Mitigation: use `forbidden_or_not_found` for unauthorized private ids.
- Repudiation: Fire-and-forget socket failures can disappear. Mitigation: emit stable `socket:error` when no ack callback is supplied.
- Elevation of privilege: Room-wide unread emits expose another user's state. Mitigation: target intended user sockets only.
</threat_model>
