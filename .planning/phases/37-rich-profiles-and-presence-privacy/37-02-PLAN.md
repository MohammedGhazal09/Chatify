# Phase 37 Plan 02 - Frontend Settings Profile And Privacy Controls

## Objective

Expose profile bio/status and presence privacy controls in Settings with clear validation, feedback, and cache updates.

## Scope

- Frontend user types and user API methods for profile and privacy updates.
- Profile mutation hook updates auth, chats, users, user search, and presence caches.
- Settings profile section with bio/status fields.
- Settings privacy controls for online status, last seen, and profile status.
- Component tests for validation, successful saves, failed saves, and email remaining account-only.

## Acceptance Criteria

- [ ] Settings can save profile bio/status and update local auth state.
- [ ] Settings can toggle online, last-seen, and status visibility.
- [ ] Validation errors prevent invalid profile saves before API calls.
- [ ] Account email remains visible only in the signed-in account summary.
- [ ] Cache invalidation updates conversation/member profile surfaces.
