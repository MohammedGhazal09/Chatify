<threat_model>
Primary assets: private attachment bytes, attachment metadata, message visibility state, chat membership, pin state, and realtime room membership.
Threats: unauthorized preview/download by non-members, attachment access after delete-for-self or delete-for-everyone, public URL or GridFS id leakage, malformed uploads causing storage or memory pressure, multipart bypass of message idempotency/status behavior, orphaned GridFS files after partial failure, and room events leaking attachment or pin metadata outside the chat.
Mitigations: reuse `loadChatForUser` and message visibility checks for every attachment and pin route, expose only sanitized summaries, never return storage ids or public URLs, mount Multer only on the canonical send route with count/size/type limits, validate file signatures and text-like formats server-side, compare attachment fingerprints for idempotency, clean up storage on downstream failure, add upload-specific rate limiting, and emit realtime events only after protected HTTP mutations succeed.
Residual risk: virus scanning and deep content inspection are explicitly outside Phase 08, so copy and tests must avoid claiming scanned or malware-safe files.
</threat_model>
