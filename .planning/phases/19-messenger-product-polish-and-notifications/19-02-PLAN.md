# Plan 19-02: Notification UI And Realtime Alert Wiring

## Objective

Wire the notification model into visible Settings controls, conversation mute affordances, in-app alerting, sound alerting, and browser notification behavior without exposing private message content.

## Tasks

1. Read `19-01-SUMMARY.md` if present, then read the notification utilities, Settings modal, socket hook, Toast provider, sidebar/list item components, conversation menu, and `chat.tsx`.
2. Replace the simple sound toggle in Settings with a notification section that shows sound preference, browser notification opt-in, permission status, unsupported/denied guidance, and muted-conversation implications.
3. Add mute/unmute controls for the selected conversation through an existing conversation action surface, plus a subtle muted indicator in the sidebar/list item.
4. Update `useChatSocket.ts` so incoming messages update caches and receipts exactly as before while routing alerts through the new preference model.
5. Ensure current foreground chat messages do not create duplicate alerts, muted conversations do not alert, and background conversations can alert with generic privacy-safe copy.
6. Use browser notifications only when opted in, permission is granted, the environment supports `Notification`, and the message is alert-eligible.
7. Use in-app toast/status alerting for background messages where appropriate, but keep copy generic and avoid message text.
8. Add tests for Settings permission states, mute toggling, sidebar muted indicator, socket alert routing, sound suppression, browser notification eligibility, and no private text leakage.

## Verification

- `cd Frontend/Chatify; npm test -- --run src/components/SettingsModal.test.tsx src/hooks/useChatSocket.test.tsx`
- `cd Frontend/Chatify; npm test -- --run src/pages/chat/components/ChatSidebar.test.tsx src/pages/chat/components/ChatListItem.test.tsx src/pages/chat/components/ConversationMoreMenu.test.tsx`
- `cd Frontend/Chatify; npm run lint`

## Success Criteria

Users can understand and control Chatify alerts from the UI, while realtime background alerts stay useful, muted when requested, and privacy-safe.

## Must Haves

- No browser notification body or title may include raw message text.
- Muted conversations must still receive realtime cache updates and unread counts.
- Permission prompts must only happen after a user action.
- Any edits to `Frontend/Chatify/src/pages/chat/chat.tsx` must be narrow integration edits and preserve unrelated local work.

## Artifacts This Phase Produces

- Settings notification controls
- Per-conversation mute UI
- Privacy-safe realtime alert routing
- Notification UI and socket tests
