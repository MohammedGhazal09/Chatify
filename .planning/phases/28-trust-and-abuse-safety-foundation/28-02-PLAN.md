# 28-02 Plan: Authorized Moderation Review And Audit Trail

## Goal

Let maintainers review reports through a server-authorized admin path with auditable decisions.

## Tasks

- [x] Add `role` to users as a server-selected admin flag.
- [x] Add `requireAdmin` middleware that loads the role from MongoDB.
- [x] Add admin report list endpoint.
- [x] Add admin report review endpoint for reviewed, dismissed, and action-taken decisions.
- [x] Persist reviewer id, review timestamp, moderation action, note, and audit trail entry.

## Acceptance

- Non-admin users receive 403 for review paths.
- Admin decisions are persisted and serialized without leaking private identity data.
