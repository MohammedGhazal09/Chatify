# Phase 40 Plan 01 - Backend Appeal And Assignment Contract

## Objective

Add report assignment and user appeal state to the moderation data model and API.

## Scope

- Extend abuse report persistence with assignment fields and embedded appeals.
- Add user endpoint to list own enforcement outcomes.
- Add user endpoint to submit an appeal for supported own enforcement outcomes.
- Add admin endpoint to assign a report to self or another admin.
- Add focused backend tests.

## Acceptance Criteria

- [x] Appeal records redact user-provided reason text.
- [x] Users can only view and appeal their own enforcement outcomes.
- [x] A user cannot create multiple open appeals for the same report.
- [x] Assignment requires server-verified admin identity.
- [x] Focused backend tests pass.
