# 47-02 Plan: Frontend Invite Management And Join UX

## Goal

Expose safe invite-link controls to authorized group/space managers and add a protected invite-join route.

## Tasks

1. Add typed invite API methods and React Query hooks.
2. Add invite-link types shared by frontend components.
3. Add invite management UI:
   - active/expired/revoked metadata rows
   - expiry and max-use presets
   - create, copy, and revoke controls
   - one-time generated link display
4. Place management entry points where users already manage groups/spaces.
5. Add `/invite/:token` protected route:
   - loading state
   - joined group redirect
   - joined space redirect
   - already-member success through the same redirect path
   - generic unavailable error
6. Update caches after successful join.
7. Add focused frontend tests and visual QA runner.

## Verification

- `cd Frontend/Chatify; npm test -- --run src/pages/chat/components/InviteLinksDialog.test.tsx src/pages/chat/components/ConversationMoreMenu.test.tsx`
- `cd Frontend/Chatify; npm run lint`
- `cd Frontend/Chatify; npm run build`
- Fallback Hercules-compatible visual QA for desktop/mobile invite management and join states.

## Risks

- Link text can overflow mobile panels.
- Join route may be confusing after login because existing public route behavior does not preserve return-to-invite automatically.
- Invite management should stay hidden in direct and encrypted conversations.
