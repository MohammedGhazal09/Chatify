# Phase 31 Plan 02 - Admin Moderation Workspace UI

## Objective

Build the protected `/admin/moderation` workspace for report triage, review notes, and enforcement actions.

## Scope

- Extend frontend moderation API types/methods.
- Add TanStack Query hook for report list/detail/review.
- Add admin moderation page with queue, filters, selected detail, audit trail, review form, forbidden/loading/empty/error states.
- Add route and admin navigation affordance.
- Add frontend tests for admin, forbidden, empty/error/loading, and review submission flows.

## Acceptance Criteria

- [ ] Admin users can filter and select reports from a privacy-safe queue.
- [ ] Selected report detail shows redacted context and audit history.
- [ ] Review form submits status, enforcement action, and note through the moderation API.
- [ ] Non-admin users see `Admin access required`.
- [ ] UI controls are keyboard-addressable and have accessible names.
