## Regression Fixed During Verification

The first current rerun of the local smoke skipped because Phase 1 CSRF enforcement caused smoke account provisioning to receive `403 CSRF token invalid or missing`.

Fix applied:

- `Frontend/Chatify/e2e/chat-delivery-reliability.spec.ts` now fetches `/api/csrf-token` and sends the matching `X-CSRF-Token` header plus `XSRF-TOKEN` cookie for local API signup provisioning.
- `Frontend/Chatify/e2e/pages/productionSmoke.ts` now bootstraps CSRF before API login for production smoke browser contexts.

Rationale: smoke helpers should follow the authenticated browser contract instead of bypassing or weakening backend CSRF protection.
