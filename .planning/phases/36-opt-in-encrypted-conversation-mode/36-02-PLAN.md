# Phase 36 Plan 02 - Frontend Encryption Helpers And Send/Display Flow

## Objective

Wire the frontend to create encrypted conversations, encrypt outgoing text locally, decrypt locally available messages, and show honest missing-secret states.

## Scope

- Chat/message TypeScript fields for encryption mode and encrypted payloads.
- Chat and message API payload support.
- Browser Web Crypto helper for AES-GCM text encryption/decryption and local secret storage.
- `useSendMessage` encrypted payload branch for `e2ee_v1` conversations.
- Message rendering for decrypted text, unavailable local secret, and decrypt failure.
- Tests for helper behavior, API payload shape, hook encrypted sends, and message rendering.

## Acceptance Criteria

- [ ] Encrypted conversation creation sends `encryptionMode: 'e2ee_v1'`.
- [ ] Encrypted send does not pass plaintext `text` to the message API.
- [ ] Messages decrypt only when the local conversation secret is present.
- [ ] Missing or invalid local secret renders an honest unavailable state.
- [ ] Standard message sending remains unchanged.
