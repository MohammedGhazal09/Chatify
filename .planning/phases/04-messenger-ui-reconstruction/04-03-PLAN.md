<threat_model>
Trust boundaries:
- Test fixtures represent private messages, auth state, and action permissions.
- Test runtime can accidentally mask missing accessible names or invalid DOM.
- Smoke evidence may rely on local auth/backend availability.

STRIDE register:
- Spoofing: Tests could assert own-message actions on wrong sender data. Mitigation: fixtures include own and other user ids and verify actions differ.
- Tampering: Tests could bypass real component callbacks. Mitigation: assert callback payloads include exact `messageId`, `chatId`, `text`, and `clientMessageId` where relevant.
- Information disclosure: Tombstone tests must ensure original deleted text is not rendered.
- Denial of service: Test setup could slow every run. Mitigation: jsdom and setupFiles only; no permanent browser automation dependency.
- Repudiation: Smoke checks could be vague. Mitigation: `04-SMOKE.md` records viewport, command/server URL, checks performed, and pass/skip result.
</threat_model>
