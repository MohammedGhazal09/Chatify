# Phase 32 Plan 03 - Frontend Settings Integration And Evidence

## Objective

Expose server-backed push/email preferences in the existing Settings modal while preserving local sound/browser alert behavior and producing focused verification evidence.

## Scope

- Notification preference API client types.
- `useNotificationPreferences` server synchronization with local fallback.
- Settings modal push/email controls, pending/error/unsupported states, and copy updates.
- Optional service worker registration helper for browser push subscription when supported.
- Frontend tests and final verification artifacts.

## Acceptance Criteria

- [ ] Settings shows push and email notification controls with privacy-safe copy.
- [ ] Push/email toggles call authenticated preference APIs and handle pending/error states.
- [ ] Existing sound and browser alert permission tests continue to pass.
- [ ] Muted conversation controls continue to update local UI and server preferences when authenticated.
- [ ] Frontend tests, lint, and build pass for changed surfaces.

## Verification

- `npm test -- useNotificationPreferences.test.tsx SettingsModal.test.tsx` from `Frontend/Chatify`.
- `npm run lint` from `Frontend/Chatify`.
- `npm run build` from `Frontend/Chatify`.
