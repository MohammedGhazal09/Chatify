# Phase 37 Plan 01 - Backend Profile And Presence Privacy Contract

## Objective

Add backend profile text fields, validation, privacy serialization, and block-aware presence behavior.

## Scope

- User model fields for `profileBio`, `profileStatus`, and `showProfileStatus`.
- Profile text normalization/validation helper.
- `PATCH /api/user/profile` current-user endpoint.
- Public identity serialization with profile fields and status visibility.
- `PATCH /api/user/privacy-settings` support for `showProfileStatus`.
- Block-aware `getOnlineUsers` contact filtering.
- Socket presence broadcasts that actively clear stale online state when visibility is hidden.
- Backend tests for validation, privacy serialization, blocked presence, and privacy-change broadcasts.

## Acceptance Criteria

- [ ] Profile bio/status persist with length and unsafe-content validation.
- [ ] Public identity responses include only approved profile fields and never include email.
- [ ] Hidden status text is not serialized to other users.
- [ ] Blocked contacts do not appear in presence snapshots.
- [ ] Hiding online status broadcasts an offline/unreachable state to authorized contacts.
