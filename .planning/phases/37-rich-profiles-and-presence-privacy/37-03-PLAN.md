# Phase 37 Plan 03 - Conversation Profile Surfaces And Realtime Privacy

## Objective

Render approved profile/status data in conversation surfaces and reconcile presence privacy across realtime updates.

## Scope

- Conversation detail contact-card profile status and bio.
- Optional compact status display in conversation header where existing layout allows.
- Presence store behavior for hidden/unavailable statuses without stale online state.
- Socket identity/profile update cache merge for profile fields.
- Tests for conversation detail rendering, status hiding, and presence store updates.

## Acceptance Criteria

- [ ] Conversation detail shows allowed bio/status without email.
- [ ] Hidden status text does not render in contact-card surfaces.
- [ ] Presence-hidden users render unavailable/offline and are not call-reachable.
- [ ] Realtime profile updates refresh member/profile caches.
- [ ] Mobile/dense layout text remains bounded.
