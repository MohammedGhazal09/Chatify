# Phase 51 Plan 02: Operations Cards And Aggregate Summaries

## Tasks

- Add moderation operations summary values to the moderation card.
- Add delivery-health status and summary values to the delivery card.
- Keep both cards navigable when summary requests are loading or fail.
- Add component tests for loaded, loading/error, and RTL rendering.

## Acceptance

- The hub never displays raw messages, emails, tokens, cookies, or provider payloads.
- The hub remains useful when one summary request fails.
- RTL and mobile layout keep text readable.

## Recommendation

Use existing hooks directly and avoid adding a backend aggregation endpoint until there are at least three admin tools with meaningful shared server-side summary needs.
