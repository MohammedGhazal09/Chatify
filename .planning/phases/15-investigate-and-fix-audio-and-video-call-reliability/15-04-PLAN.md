# Plan 15-04: Acceptance, Regression, Evidence, And Production Decision

<objective>
Run the complete Phase 15 acceptance matrix, record local and production evidence honestly, verify messenger regressions, and finish with a clear call readiness decision.
</objective>

## Context

This is the final gate. It should not introduce large new implementation work except targeted fixes for failures uncovered by the acceptance matrix. Production readiness must be passed or explicitly blocked with concrete reasons.

## Artifacts this phase produces

- `15-CALL-ACCEPTANCE.md` with command results, local fake-media evidence, production smoke pass/block, TURN readiness, screenshots/traces, and residual risks.
- Updated `15-FAILURE-REPORT.md` mapping each finding to fix status and verification evidence.
- Final verification command log summaries.

<must_haves>
- Local two-account fake-media audio call passes.
- Local two-account fake-media video call passes.
- Production call readiness is passed only with configured production smoke and TURN readiness, otherwise blocked.
- Security/privacy evidence is recorded.
- Messenger regression checks pass or known non-call failures are documented separately.
</must_haves>

<threat_model>
Assets: readiness claim, production smoke accounts, production origins, auth cookies, call session data, TURN readiness, screenshots/traces, and final user-facing reliability.
Threats: claiming success from local-only tests, leaking production credentials or user identifiers in artifacts, ignoring TURN absence, masking messenger regressions, and leaving stale call sessions after acceptance tests.
Mitigations: explicit production env gate, redacted artifact writer, no local-origin fallback, TURN readiness check, cleanup after every call test, and final evidence review.
Residual risk: production smoke still uses controlled test accounts and cannot represent every user network; residual NAT/browser risks must be documented.
</threat_model>
