<threat_model>
Trust boundaries:
- Socket payloads cross from realtime transport into client cache state.
- Optimistic client-generated ids cross into server correlation and then back into persisted payloads.
- Cached unread counts cross from server truth into user-visible badges.

STRIDE register:
- Spoofing: Client-supplied sender remains in frontend payloads. Mitigation: remove sender from `messageApi.createMessage()` and rely on server session identity.
- Tampering: Socket events could overwrite newer local state. Mitigation: shared helpers use canonical ids and ranked status patches.
- Information disclosure: Deleted/tombstoned payloads could keep old text in cache. Mitigation: tombstone helper replaces content with canonical server tombstone.
- Denial of service: Duplicate socket/refetch events can grow message arrays. Mitigation: helper dedupes by `_id` and `clientMessageId`.
- Repudiation: Failed send rollback could erase unrelated messages. Mitigation: mark failed optimistic item instead of restoring whole old snapshots.
</threat_model>
