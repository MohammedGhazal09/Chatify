# Phase 34 Plan 01 - Backend Search Contract And Jump Context

## Objective

Extend selected-conversation search with sender, date, type, asset, and link filters, then add an authorized message-context endpoint for jump-to-result.

## Scope

- Search filter normalization in `messageState.mjs`.
- Filtered search behavior in `messageController.mjs`.
- Message context endpoint and route.
- Message model indexes where useful for bounded filtered queries.
- Backend tests for authorization, visibility, filters, and context windows.

## Acceptance Criteria

- [ ] Sender, date, type, and text filters are validated and applied.
- [ ] Media/file/voice filters use active attachment metadata.
- [ ] Link filters match visible URL text only.
- [ ] Empty text queries require at least one narrowing filter.
- [ ] Message context returns an authorized chronological window around the target message.
- [ ] Non-members and hidden/deleted messages remain private.
