# Phase 42 Plan 01 - Backend Contact Request Contract

## Objective

Add the backend request lifecycle and enforce the request gate for new standard direct chats.

## Scope

- Add a `ContactRequest` Mongoose model.
- Add request serializer/helpers using public identity only.
- Add list/create/accept/decline/cancel routes under `/api/chat/contact-requests`.
- Update direct chat creation so new standard direct chats require accepted trust, while existing direct chats still return normally.
- Emit privacy-safe request lifecycle socket events where useful.
- Add backend tests for pending, accept, decline, cancel, privacy, blocked users, idempotency, and existing-chat continuation.

## Acceptance Criteria

- [x] First non-contact standard direct attempt returns pending request data and does not create a chat.
- [x] Repeating the same request reuses the pending request.
- [x] Accepting an incoming request creates or returns one direct chat.
- [x] Decline and cancel never create a chat.
- [x] Request lifecycle endpoints reject unauthorized users and blocked/self/missing targets safely.
- [x] Response and socket payloads contain no email fields.
