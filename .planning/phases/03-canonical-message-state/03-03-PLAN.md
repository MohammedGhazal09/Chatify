<threat_model>
Trust boundaries:
- Cursor parameters cross from client query strings into database range queries.
- User-visible message history crosses from persisted private records into chat UI state.
- Chat sidebar latest-message projection crosses from shared chat documents into per-user views.

STRIDE register:
- Tampering: Malformed cursors can cause unstable queries or errors. Mitigation: validate cursor shape and limit bounds before querying.
- Information disclosure: Delete-for-self messages or unauthorized messages can appear in history, unread counts, or sidebar latest message. Mitigation: shared visibility filter across history, unread, and latest-message projection.
- Denial of service: Excessive `limit` or deep offset queries can stress MongoDB. Mitigation: bounded limit and cursor range queries instead of skip.
- Repudiation: Validation mismatches let invalid messages pass one layer and fail another. Mitigation: shared constants/helpers and backend/frontend tests for boundary values.
</threat_model>
