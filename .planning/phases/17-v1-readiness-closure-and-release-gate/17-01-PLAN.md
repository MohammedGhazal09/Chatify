# Plan 17-01: Evidence Inventory And Blocker Matrix

## Objective

Create the initial v1 readiness evidence matrix from current planning artifacts and identify every release-blocking gap before running additional commands.

## Tasks

1. Read `17-SPEC.md`, `17-CONTEXT.md`, `ROADMAP.md`, `STATE.md`, `REQUIREMENTS.md`, and readiness artifacts from Phases 1, 10, 10.1, 14, and 15.
2. Create or update `17-V1-READINESS.md` with requirement rows, evidence paths, status (`passed`, `blocked`, `failed`, `not_checked`), and release impact.
3. Mark missing production smoke env, missing local Phase 15 smoke env, missing deployed evidence, and any unresolved requirement as release-blocking.

## Verification

- `17-V1-READINESS.md` exists.
- No row claims readiness without an evidence path or command result.
- Artifact contains no secrets or full account identifiers.

## Success Criteria

The phase has a release blocker matrix before any final readiness decision is made.
