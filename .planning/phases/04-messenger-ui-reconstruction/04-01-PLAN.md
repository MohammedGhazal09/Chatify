<threat_model>
Trust boundaries:
- Private message content crosses from Query cache into presentational components.
- User identity and presence cross from Zustand stores into sidebar/header components.
- Message action intent crosses from UI controls back into existing mutations.

STRIDE register:
- Spoofing: A presentational component could accept arbitrary current-user identity. Mitigation: current-user checks stay centralized in `chat.tsx` or typed props derived from `useAuthStore`.
- Tampering: Component extraction could bypass canonical Query helpers. Mitigation: extracted components receive already-derived messages and callbacks; they do not mutate Query cache directly.
- Information disclosure: Deleted-for-everyone messages could render stale text. Mitigation: `MessageBubble` preserves tombstone handling from Phase 3 types and current inline bubble logic.
- Denial of service: New scroll listeners could accumulate during extraction. Mitigation: keep scroll listener cleanup in `MessageList` and verify with lint/build.
- Elevation of privilege: Message action components could show edit/delete for non-owned messages. Mitigation: pass explicit `isOwnMessage`/allowed-action props derived by the orchestrator.
</threat_model>
