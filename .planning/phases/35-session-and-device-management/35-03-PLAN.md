# Phase 35 Plan 03 - Frontend Session Management UI

## Objective

Expose active sessions and revocation actions in Settings with privacy-safe labels, current-session state, loading/error states, and logout-everywhere behavior.

## Scope

- Auth API methods and types.
- Session management hook.
- Settings UI section.
- Tests for render, revoke, logout everywhere, and privacy boundaries.

## Acceptance Criteria

- [ ] Settings lists active sessions with current-session state.
- [ ] Non-current sessions expose a revoke action.
- [ ] Log out everywhere clears local auth state after server success.
- [ ] UI does not render raw IPs, user agents, tokens, hashes, cookies, or emails.
- [ ] Tests cover loading, error, empty, and success states.

## Verification

- `npm test -- SettingsModal.test.tsx useAuthQuery.test.tsx authApi.test.ts` from `Frontend/Chatify`.
- `npm run lint` and `npm run build` from `Frontend/Chatify`.
