# Phase 39 Plan 02 - Deletion Request And Retention Contract

## Objective

Add authenticated, CSRF-protected account deletion request and cancellation endpoints with clear retention behavior.

## Scope

- Create pending deletion requests with scheduled date and retention summary.
- Return existing pending request idempotently.
- Cancel pending deletion requests.
- Do not delete account data immediately.
- Add focused backend deletion tests.

## Acceptance Criteria

- [x] Deletion request endpoint creates or returns one pending request per user.
- [x] Cancellation endpoint cancels only the requester's pending request.
- [x] Retention/anonymization/tombstone summary is returned to the user.
- [x] Audit records omit raw private payloads.
- [x] Focused backend deletion tests pass.
