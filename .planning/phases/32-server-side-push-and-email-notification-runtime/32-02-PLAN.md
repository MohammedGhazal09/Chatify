# Phase 32 Plan 02 - Provider Worker And Sanitized Delivery Outcomes

## Objective

Implement a bounded notification outbox processor that sends or dry-runs generic email/push notifications and records sanitized delivery results.

## Scope

- Generic notification template helper.
- Email notification provider reusing the existing Brevo transport boundary.
- Push notification provider boundary with VAPID configuration checks and safe dry-run behavior.
- Outbox processor with bounded batch size, retry count, backoff, and sanitized errors.
- Startup worker gating that is disabled in tests and configurable in runtime.

## Acceptance Criteria

- [ ] Provider payloads use generic copy and do not include message text or attachment names.
- [ ] Test/local dry-run delivery marks jobs sent without network calls.
- [ ] Missing production provider config fails closed with sanitized failure state.
- [ ] Failed jobs retry with bounded backoff and stop after max attempts.
- [ ] Delivery logs/results exclude emails, push endpoints, tokens, cookies, and provider secrets.
