# Phase 32 Plan 01 - Backend Preferences And Outbox Contract

## Objective

Add server-owned notification preferences, push subscription storage, and a durable notification outbox that can be populated safely from new message creation.

## Scope

- User notification preference schema and serializers.
- Preference API routes for fetch/update and push subscription registration/removal.
- Notification outbox model with dedupe keys and sanitized payload fields.
- Message creation enqueue hook that only runs for newly persisted messages.
- Backend tests for preferences, mute/block/unsubscribe suppression, and outbox dedupe.

## Acceptance Criteria

- [ ] Authenticated users can fetch default notification preferences.
- [ ] Preference updates are CSRF-protected and reject invalid muted chat IDs or preview modes.
- [ ] Push subscriptions are stored without logging endpoint values.
- [ ] New messages enqueue eligible email/push jobs only once per recipient/message/channel.
- [ ] Sender, muted chat, blocked direct peer, unsubscribed email, and unavailable push subscription cases are skipped before enqueue.
- [ ] Idempotent message retries do not add outbox jobs.
