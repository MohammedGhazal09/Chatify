# 47-01 Plan: Backend Invite Link Authority

## Goal

Add secure invite-link persistence and APIs for groups and spaces.

## Tasks

1. Add `inviteLinkModel.mjs` with hashed tokens, target type/id, creator, expiry, revocation, use count, max uses, and timestamps.
2. Add invite utilities for token generation, hashing, expiry/max-use validation, serialization, and active-state checks.
3. Add invite controller endpoints:
   - create group invite
   - list group invites
   - create space invite
   - list space invites
   - revoke invite
   - join invite by token
4. Wire invite routes under the existing protected API surface.
5. Enforce authorization:
   - group invites require `groupAdmin`
   - space invites require owner/admin
   - direct and encrypted targets rejected
6. Implement join behavior:
   - already-member success
   - group member add with cap/block checks
   - space member add with cap/block checks and channel membership sync
   - invite use count increments only for new successful joins
7. Add focused backend tests.

## Verification

- `cd Backend/Chatify; npm test -- test/invite/invite-links.test.mjs test/chat/chat.group.test.mjs test/space/space.messaging.test.mjs`

## Risks

- Token leakage through logs or list responses.
- Race conditions around usage cap and target member cap.
- Accidentally reopening direct or encrypted conversation onboarding boundaries.
