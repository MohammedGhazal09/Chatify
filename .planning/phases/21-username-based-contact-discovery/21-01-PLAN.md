# Plan 21-01: Backend Username Direct-Chat Contract And Privacy-Safe Lookup

## Objective

Replace direct-chat email lookup with validated username lookup and expose an exact public identity lookup endpoint.

## Tasks

1. Add public lookup controller using `validateUsername` and public identity serialization.
2. Mount `GET /api/user/lookup/:username` behind `protect`.
3. Change `createChat` to require `targetUsername`, reject invalid/missing values, resolve by normalized username, and stop using `targetEmail`.
4. Preserve directKey idempotency and chat socket notification behavior.
5. Update backend tests for valid username, repeat submit, invalid username, self target, missing username, email payload rejection, lookup no-email, and lookup contact privacy.
