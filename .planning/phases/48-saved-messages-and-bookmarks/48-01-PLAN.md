# 48-01 Plan: Backend Saved Message Authority

## Goal

Add private, per-user saved-message persistence and APIs that reuse existing message visibility rules.

## Requirements

- V2-SAVED-01
- V2-SAVED-02
- V2-SAVED-03
- V2-SAVED-05
- V2-SAVED-06

## Tasks

1. Add `Backend/Chatify/Models/savedMessageModel.mjs`:
   - fields: `user`, `message`, `chat`, `savedAt`
   - unique index `{ user: 1, message: 1 }`
   - list index `{ user: 1, savedAt: -1, _id: -1 }`
2. Extend `Backend/Chatify/Controller/messageController.mjs` with saved-message helpers:
   - load visible save target by message id and requester
   - decorate serialized messages with `savedByRequester` and `savedAt`
   - serialize saved-list rows with safe message, chat, sender, and saved timestamp metadata
3. Add controller handlers:
   - `listSavedMessages`
   - `saveMessage`
   - `unsaveMessage`
4. Wire routes in `Backend/Chatify/Routes/messageRouter.mjs`:
   - `GET /api/message/saved`
   - `POST /api/message/:messageId/save`
   - `DELETE /api/message/:messageId/save`
5. Enforce privacy and authorization:
   - requester must be authenticated
   - requester must be a current chat member
   - message must pass `canUserSeeMessage`
   - deleted-for-everyone messages cannot be saved or returned as usable saved entries
   - delete-for-self-hidden messages cannot be saved or listed
   - encrypted saved rows use generic safe preview metadata
6. Add focused backend tests in `Backend/Chatify/test/message/message.saved.test.mjs`.

## Risks

- Route shadowing if `/saved` is placed after parameterized routes.
- Accidentally leaking hidden/deleted/encrypted content in saved-list payloads.
- Mutating shared message state instead of per-user saved state.

## Threat Model

- **Private-message leakage:** Saved-list endpoints could reveal messages from chats the requester no longer belongs to. Mitigation: re-run membership and visible-message filters during every list/save response.
- **Cross-user state pollution:** One user's save could affect another user if stored on `Messages`. Mitigation: use dedicated `SavedMessage` rows keyed by user and message.
- **Encrypted plaintext exposure:** Saved-list serialization could show decrypted text. Mitigation: backend never receives decrypted plaintext; frontend must render generic encrypted preview.
