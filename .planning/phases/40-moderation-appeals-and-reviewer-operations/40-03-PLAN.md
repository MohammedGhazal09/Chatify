# Phase 40 Plan 03 - Frontend Appeals And Reviewer Operations UI

## Objective

Expose user appeals in Settings and admin assignment/appeal/metrics workflows in the moderation workspace.

## Scope

- Extend moderation API types and hooks.
- Add Settings account-safety appeal controls.
- Add admin operations summary, assignment controls, appeal resolution, and enforcement history.
- Add focused frontend tests.

## Acceptance Criteria

- [x] Users can view appealable enforcement outcomes and submit appeals from Settings.

- [x] Loading, empty, success, and error states are accessible.
- [x] Focused frontend tests, lint, and build pass.
