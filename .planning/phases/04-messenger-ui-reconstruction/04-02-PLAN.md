<threat_model>
Trust boundaries:
- Message content and action permissions cross from app state into UI controls.
- Offline/session/socket state crosses into composer availability.
- Destructive actions cross from menu controls into delete mutations.

STRIDE register:
- Spoofing: UI may show own-message actions on non-owned messages. Mitigation: derive `isOwnMessage` and allowed actions before rendering `MessageActionMenu`.
- Tampering: Retry could change failed-message identity. Mitigation: retry passes the existing failed message `clientMessageId`.
- Repudiation: Delete actions could be ambiguous. Mitigation: destructive copy distinguishes delete-for-self and delete-for-everyone.
- Information disclosure: Tombstone UI could expose original text. Mitigation: deleted-for-everyone state renders neutral tombstone only.
- Denial of service: Static emoji picker can inflate chat bundle. Mitigation: lazy-load picker on demand.
- Elevation of privilege: Hidden hover-only controls can be inaccessible and misleading. Mitigation: explicit kebab controls and keyboard paths for all core actions.
</threat_model>
