# 28-01 Plan: Backend Abuse Report Contract And Privacy Redaction

## Goal

Create a report intake contract that users can call safely for users, messages, and conversations.

## Tasks

- [x] Add `AbuseReport` model with target fields, status, context snapshots, and audit trail fields.
- [x] Add report creation controller with target validation and membership checks.
- [x] Add privacy redaction for emails, bearer tokens, secret/password/cookie fields, and long token-like strings.
- [x] Add moderation router and mount it behind auth and CSRF.
- [x] Add report-specific rate limiter.

## Acceptance

- Report creation is authenticated, CSRF-protected, membership-checked, and redacted.
