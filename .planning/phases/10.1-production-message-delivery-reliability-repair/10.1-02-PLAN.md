<threat_model>
- T-10.1-05: Rapid UI events submit the same draft more than once with different `clientMessageId` values.
- T-10.1-06: Split cache ownership creates duplicate rendered rows or false status.
- T-10.1-07: Recipient delivery ack fires before the message is actually cached.
- T-10.1-08: Reconnect/refetch duplicates missed messages instead of reconciling them.

Mitigations: same-draft guard, centralized Query cache writes, client-id-first merge tests, recipient ack after cache upsert, absolute unread updates, and reconnect merge tests.
</threat_model>
