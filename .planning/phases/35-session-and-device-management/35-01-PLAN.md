# Phase 35 Plan 01 - Backend Session Inventory And Revocation

## Objective

Extend the existing refresh session contract with safe device metadata, active-session listing, individual revocation, and logout-everywhere APIs.

## Scope

- Session model metadata fields.
- Session serialization helpers.
- Token issue/rotate metadata capture.
- Auth routes/controllers for session list, revoke one, and revoke all.
- Backend tests for metadata privacy, current-session identification, and revocation behavior.

## Acceptance Criteria

- [ ] Active sessions list excludes revoked/expired rows.
- [ ] Responses include safe labels and timestamps only.
- [ ] Users can revoke a non-current session.
- [ ] Log out everywhere revokes all active sessions and clears current cookies.
- [ ] Tests prove raw metadata and hashes are not exposed.
