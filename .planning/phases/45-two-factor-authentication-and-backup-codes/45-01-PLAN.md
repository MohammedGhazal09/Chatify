# Phase 45 Plan 45-01: Backend 2FA Authority

## Goal

Add the backend persistence, TOTP helpers, challenge flow, and protected Settings endpoints required for secure 2FA.

## Scope

- Add encrypted TOTP secret storage and hashed backup-code fields to the user model.
- Add a short-lived 2FA login challenge model.
- Add TOTP, encryption, backup-code, and challenge-token helpers.
- Modify login to return a challenge for 2FA-enabled local accounts without cookies.
- Add 2FA status/setup/confirm/disable/regenerate/challenge verification endpoints.
- Add backend tests for setup, login challenge, cookie issuance, backup-code single use, and disable/regenerate guardrails.

## Acceptance

- Non-2FA auth lifecycle tests still pass.
- 2FA-enabled password login returns a challenge and does not create a session.
- Challenge verification creates the normal access/refresh cookies.
- Invalid/replayed/expired challenges are rejected.
- Backup codes are single-use and are never stored in plaintext.

Completed locally on 2026-06-30.
