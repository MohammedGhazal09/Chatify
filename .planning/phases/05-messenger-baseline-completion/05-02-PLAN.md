<threat_model>
Trust boundaries:
- URL query params and localStorage values are untrusted browser state.
- Auth refresh failure crosses from Axios transport into private UI cleanup.
- Presence/typing maps are transient realtime state derived from authorized sockets.
- Playwright smoke fixtures simulate private chats and should not leak into production config.

STRIDE register:
- Spoofing: A user can put another chat id in `?chatId=`. Mitigation: validate against loaded accessible chats before selection.
- Tampering: LocalStorage can contain another user's prior chat id. Mitigation: per-user key and accessible-chat validation.
- Information disclosure: Stale messages could render after auth loss. Mitigation: clear Query cache and view/search state on logout or auth-expired event.
- Information disclosure: Failed contact lookup could show presence. Mitigation: presence display remains based only on existing chat members.
- Denial of service: Auth-expired event handlers could accumulate. Mitigation: hook effect cleanup and one app-level listener.
- Repudiation: Smoke evidence could be vague. Mitigation: write exact commands, viewport sizes, and pass/skip reasons to `05-SMOKE.md`.
</threat_model>
