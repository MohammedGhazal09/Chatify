# Plan 15-01: Investigation, Failure Report, And Test Harness

<objective>
Reproduce and document the current call failures before implementation, then create deterministic unit and local fake-media test harnesses that prove Phase 15 fixes are necessary and measurable.
</objective>

## Context

This is the investigation-first gate for Phase 15. It must not ship call implementation changes by itself. It creates failing or currently blocked evidence and prepares the tests that later plans must make pass.

## Context Decisions Covered

D-01 D-02 D-03 D-04 D-05 D-06 D-07 D-08 D-09 D-10 D-11 D-12 D-13 D-14 D-15 D-16 D-17 D-18 D-19 D-20 D-21 D-40 D-41 D-42 D-43 D-44 D-45 D-46 D-47 D-48 D-49 D-50 D-51 D-52 D-53 D-56 D-57 D-58

## Artifacts this phase produces

- `15-FAILURE-REPORT.md` with reproduced findings, command output summaries, screenshots/traces where applicable, and fix-to-finding placeholders.
- A local Phase 15 two-account fake-media Playwright harness.
- Updated frontend unit tests that expose current video fallback and ICE candidate timing defects.
- Backend socket/session test gaps identified or codified as failing tests.

<must_haves>
- Investigation must precede implementation.
- Failing or blocked evidence must be written before claiming a fix.
- Local full-stack fake-media acceptance must use two authenticated accounts and must not rely on production.
- Production smoke must remain explicit and must block if env or TURN readiness is missing.
- No implementation step may overwrite unrelated work in `Frontend/Chatify/src/pages/chat/chat.tsx`.
</must_haves>

<threat_model>
Assets: authenticated call sessions, chat membership, browser media permissions, production smoke credentials, TURN configuration, and privacy-sensitive call diagnostics.
Threats: false readiness claims, leaking SDP/ICE/cookies/tokens/emails in artifacts, trusting local smoke as production proof, and test fixtures bypassing auth in ways production cannot.
Mitigations: use explicit smoke env gates, redact artifacts, keep socket identity server-verified, and record blockers rather than soft-passing unavailable production/TURN checks.
Residual risk: local fake media proves call control and signaling flow, not real-world NAT traversal; production/TURN evidence remains required.
</threat_model>
