## Goal

Add a protected, metadata-only admin diagnostics API for recent message delivery health.

## Tasks

1. Add a focused delivery-health utility:
   - create `Backend/Chatify/Utils/deliveryHealth.mjs`
   - normalize `window` query values: `1h`, `24h`, `7d`; default `24h`
   - expose `buildDeliveryHealthPayload({ windowKey, now })`
   - compute total, sent, delivered, read, stale sent, stale delivered, delivery rate, and read rate
   - exclude `messageType: "call"`
   - aggregate risk rows by `chatId` and limit to 10
   - summarize `NotificationOutbox` by status/channel without serializing payload bodies
   - include `getSocketOperationalStatus()` output and tolerate uninitialized sockets
2. Add an admin controller:
   - create `Backend/Chatify/Controller/adminController.mjs`
   - add `getDeliveryHealth`
   - return `400` for invalid windows and `200` with the diagnostics payload for valid windows
3. Add admin routing:
   - create `Backend/Chatify/Routes/adminRouter.mjs`
   - wire `GET /delivery-health` behind `moderationReviewLimiter` and `requireAdmin`
   - mount `/api/admin` in `Backend/Chatify/app.mjs` with `protect` and `csrfProtection`
4. Add backend tests in `Backend/Chatify/test/admin/delivery-health.test.mjs`:
   - unauthenticated/non-admin/admin access
   - exact lifecycle aggregate counts/rates
   - stale sent and stale delivered thresholds
   - risk row ordering and limit
   - direct/group/space-channel metadata
   - outbox status/channel counts
   - missing Socket.IO initialization behavior
   - privacy guard against seeded emails, message text, token-shaped strings, encrypted payloads, attachment names, and provider payload text

## Risks

- Accidentally populating or serializing private content in an admin diagnostics response.
- Misclassifying call activity as ordinary message delivery failure.
- Querying all historical messages instead of the bounded selected window.

## Threat Model

- **Admin endpoint exposure:** Non-admin users could access operational metadata. Mitigation: route through `protect` plus `requireAdmin`, with backend tests for 401/403.
- **Private content leakage:** Delivery health could expose text, email, encrypted payloads, attachment filenames, or provider payload bodies. Mitigation: metadata-only aggregation and seeded sensitive-string response tests.
- **Operational write regression:** A dashboard query could mutate message or outbox state. Mitigation: implement read-only aggregation and include no update/write calls in the utility.
