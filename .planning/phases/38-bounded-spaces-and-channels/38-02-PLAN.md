# Phase 38 Plan 02 - Channel Messaging And Realtime Reliability

## Objective

Wire space channels into the existing message, unread, socket, notification, attachment, reaction, and moderation contracts.

## Scope

- Extend chat/message authorization to recognize space-channel conversations.
- Ensure channel messages use existing send/list/read/unread/reaction/attachment behavior.
- Emit channel create/member update/message events only to authorized members.
- Update notification eligibility and templates with privacy-safe channel context.
- Extend abuse report creation and reviewer serialization with redacted space/channel context.
- Regression tests proving direct/group chat behavior remains intact.

## Acceptance Criteria

- [x] Channel members can send and read channel messages through existing APIs.
- [x] Non-members and removed members cannot access channel messages, attachments, unread counts, or realtime events.
- [x] Reactions, read receipts, and unread updates work for channel messages.
- [x] Notification and report context remains privacy-safe.
- [x] Direct and group chat tests still pass.
