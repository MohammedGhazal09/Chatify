# Phase 36 Plan 03 - Limitations, Notifications, Search, And Safety Copy

## Objective

Make encrypted-mode limitations explicit across search, notifications, detail surfaces, and moderation/reporting-related copy.

## Scope

- Encrypted notification template and outbox tests.
- Backend search rejection for encrypted conversations.
- Frontend search disabled state for encrypted conversations.
- Composer attachment-disabled state for encrypted conversations.
- Detail/header copy that avoids unsupported claims.
- Grep guards for unsupported encryption marketing copy.

## Acceptance Criteria

- [ ] Encrypted notifications use generic encrypted-message copy only.
- [ ] Backend search for encrypted conversations returns a clear unsupported response.
- [ ] Frontend search control explains server search is unavailable for encrypted conversations.
- [ ] Plaintext attachment controls are disabled in encrypted conversations.
- [ ] Copy guard finds no Signal-grade, hidden-metadata, or guaranteed-recovery claims.
