# Plan 46-02: Composer Suggestions And Mention Rendering

## Objective

Add the user-facing mention workflow to the existing composer and message bubble components while preserving optimistic sends, retry behavior, and responsive layout.

## Must Haves

- Show member suggestions for active `@` tokens in group chats and space channels only.
- Insert `@username ` at the textarea caret and send mention ids with the existing message payload.
- Carry mention snapshots through optimistic sends, retries, and canonical cache merge.
- Render serialized mention tokens inline in message bubbles.
- Emphasize the current user's mention without changing message ordering or status display.

## Tasks

1. Add mention types and a small frontend mention utility.
2. Extend message API payload handling for JSON and multipart sends.
3. Extend send mutation variables, optimistic message creation, and retry payloads.
4. Add composer suggestion UI and keyboard/click insertion behavior.
5. Add mention rendering to message bubbles and pass current-user context through `MessageList`.
6. Add focused component/hook tests.
