# Plan 16-04: Acceptance Evidence, Privacy Scan, And Regression Gate

<objective>
Prove Phase 16 end to end with backend, frontend, fixture guard, privacy scan, lint/build, and a local two-account Playwright flow using generated profile image files.
</objective>

## Context

This is the final acceptance pass. It does not introduce new feature scope; it closes evidence gaps, verifies cross-user visibility, and writes a redacted acceptance artifact. Production E2E is out of scope for Phase 16.

## Context Decisions Covered

D-11 D-19 D-20 D-22 D-35 D-36 D-37 D-38 D-39

## Artifacts this plan produces

- Local two-account Playwright profile picture acceptance spec.
- Generated local test image fixtures.
- Redacted Phase 16 acceptance evidence artifact.
- Final regression and privacy scan results.

<must_haves>
- Acceptance must use real generated image files from disk, not static production demo avatars.
- Missing local env/accounts must produce an honest blocked artifact with exact requirements.
- Evidence must be redacted and must not include tokens, cookies, emails, storage ids, hashes, or raw image bytes.
- Production readiness claims are not made in this phase.
</must_haves>

<threat_model>
Assets: local account credentials, uploaded profile image bytes, auth cookies, storage metadata, Playwright traces/screenshots, and acceptance reports.
Threats: leaking credentials or cookies in artifacts, mistaking local acceptance for production readiness, relying on fixtures instead of real upload flow, and missing cross-user propagation defects.
Mitigations: explicit local env gates, redacted evidence writer or manual redaction checklist, generated image fixtures, two-browser-context flow, and final storage/privacy scans.
Residual risk: production deployment can still differ from local due CORS, cookies, storage, or deployment config; Phase 16 records local acceptance only.
</threat_model>
