# Plan 46-01: Backend Mention Metadata And Validation

## Objective

Persist and serialize validated mention metadata for standard group chats and space channels without weakening message privacy, authorization, attachments, replies, sockets, or idempotency.

## Must Haves

- Add bounded `mentions` metadata to messages with public snapshots only.
- Validate mention targets against current chat membership.
- Require visible `@username` text for every submitted mention target.
- Reject mention metadata in direct chats and encrypted conversations.
- Include mention target set in idempotency comparison.
- Cover group and space paths with backend tests.

## Tasks

1. Add a mention subdocument schema and serializer support.
2. Add a backend utility for normalizing mention ids, resolving member snapshots, visible-token checks, and mention fingerprints.
3. Wire mention resolution into standard message creation before attachment storage.
4. Update idempotency lookup and duplicate handling to include mention fingerprints.
5. Add focused request tests for valid group/space mentions, invalid targets, hidden mentions, and idempotency conflicts.
