# Phase 39 Plan 01 - Backend Account Export And Privacy Audit

## Objective

Add a server-authorized privacy export endpoint and audit model without storing raw export payloads.

## Scope

- Add privacy request/audit persistence.
- Add privacy export serialization with explicit allowlists.
- Include account/profile/settings, authorized chats/spaces/channels, visible messages, attachment metadata/download references, filed reports, and encrypted limitation metadata.
- Exclude unauthorized records and private/sensitive fields.
- Add focused backend tests.

## Acceptance Criteria

- [x] Authenticated export endpoint returns a portable JSON payload.
- [x] Export excludes unauthorized chats/messages and private/sensitive fields.
- [x] Encrypted records include limitation metadata without fabricated plaintext.
- [x] Export audit record stores metadata/counts only.
- [x] Focused backend export tests pass.
