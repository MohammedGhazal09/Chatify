# Phase 31 Plan 01 - Backend Enforcement Contract

## Objective

Extend the Phase 28 moderation API so review actions apply durable, scoped enforcement and expose privacy-safe report detail data.

## Scope

- Abuse report model/action enum and serialized response.
- Admin report list/detail/review controller behavior.
- User moderation state for warnings and active messaging restrictions.
- Message send guard for active moderation restrictions.
- Backend tests for authorization, detail access, redaction, enforcement, and audit trail.

## Acceptance Criteria

- [ ] Admin list/detail responses include privacy-safe reporter/reported labels and computed priority.
- [ ] Non-admin users cannot access list, detail, or review endpoints.
- [ ] `restricted` review blocks reported user message sends while active.
- [ ] `restriction_lifted` clears the active restriction.
- [ ] `content_removed` tombstones the reported message when a message exists.
- [ ] Audit trail and report enforcement snapshots are persisted.
