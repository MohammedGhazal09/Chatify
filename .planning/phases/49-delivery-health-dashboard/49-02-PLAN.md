# 49-02 Plan: Frontend Admin Delivery Health Dashboard

## Goal

Add a responsive, localized admin dashboard that renders delivery-health diagnostics with clear loading, empty, populated, error, refresh, and restricted states.

## Requirements

- V2-DELIVERY-HEALTH-01
- V2-DELIVERY-HEALTH-02
- V2-DELIVERY-HEALTH-03
- V2-DELIVERY-HEALTH-04
- V2-DELIVERY-HEALTH-05
- V2-DELIVERY-HEALTH-06
- UI-05
- TEST-03

## Tasks

1. Add typed frontend API and hook:
   - create `Frontend/Chatify/src/api/deliveryHealthApi.ts`
   - create `Frontend/Chatify/src/hooks/useDeliveryHealth.ts`
   - expose `deliveryHealthQueryKey(windowKey)` and `useDeliveryHealth(windowKey)`
   - keep query disabled unless the authenticated user has `role === "admin"`
2. Add the admin page:
   - create `Frontend/Chatify/src/pages/admin/AdminDeliveryHealth.tsx`
   - use `chat-theme-root`, `useChatTheme`, `useLocale`, and `useAuthStore`
   - render non-admin restricted state without calling the diagnostics query
   - render header, generated timestamp, 1h/24h/7d segmented control, icon refresh button, summary metric panels, risk rows, runtime panel, outbox panel, empty state, loading state, and error state
   - use lucide icons and native buttons only
3. Wire routing:
   - lazy-load `AdminDeliveryHealth` in `Frontend/Chatify/src/App.tsx`
   - add protected route `/admin/delivery-health`
4. Add localization:
   - add English and Arabic delivery-health keys in `Frontend/Chatify/src/i18n/locales.ts`
   - include labels for title, eyebrow, refresh, windows, metrics, runtime, outbox, empty, error, restricted, and risk rows
5. Add frontend tests:
   - `Frontend/Chatify/src/api/deliveryHealthApi.test.ts`
   - `Frontend/Chatify/src/hooks/useDeliveryHealth.test.tsx`
   - `Frontend/Chatify/src/pages/admin/AdminDeliveryHealth.test.tsx`
   - cover API params, admin query enablement, non-admin access, loading, empty, populated, error, refresh, and Arabic RTL labels

## Risks

- Leaking sensitive diagnostics through frontend labels or test fixtures.
- Querying the API for non-admin users because the frontend trusted only the route guard.
- Creating a generic card-heavy dashboard instead of the dense operations console specified in `49-UI-SPEC.md`.

## Threat Model

- **Client-side role spoofing:** A user can edit frontend state locally. Mitigation: frontend disables the query for UX, but backend `requireAdmin` remains the authority.
- **Rendered sensitive data:** Test fixtures could accidentally include private strings and render them. Mitigation: component tests seed sensitive values and assert they are absent.
- **Localization drift:** English-only labels could break Phase 41 expectations. Mitigation: add English and Arabic translation keys and RTL test coverage.
