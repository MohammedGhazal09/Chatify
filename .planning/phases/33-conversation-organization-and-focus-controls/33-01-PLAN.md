# Phase 33 Plan 01 - Backend Organization Contract

## Objective

Persist per-user conversation organization state and expose it through requester-scoped chat APIs.

## Scope

- Conversation organization model and helpers.
- Chat serialization with `organizationState`.
- Protected update endpoint for mute/archive/pin/favorite.
- Socket event for same-user organization updates.
- Backend tests for per-user isolation, membership enforcement, mute sync, and sorting.

## Acceptance Criteria

- [ ] Non-members cannot update organization state.
- [ ] Archive, pin, and favorite state applies only to the requester.
- [ ] Mute updates notification preferences used by notification delivery.
- [ ] Serialized chat payloads include `organizationState` and no private emails.
- [ ] Pinned chats sort before unpinned chats.
