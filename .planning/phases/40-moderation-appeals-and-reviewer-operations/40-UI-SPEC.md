# Phase 40 UI Spec - Appeals And Reviewer Operations

## User Settings Surface

- Add an "Account safety" section to Settings.
- Show appealable enforcement outcomes with action, status, review date, and safe summary.
- Provide a small appeal form only when no open appeal exists.
- Show appeal status as open, under review, accepted, or rejected.
- Avoid reporter names, private messages, emails, and raw report internals.

## Admin Moderation Surface

- Add an operations summary strip with open reports, open appeals, unassigned work, assigned-to-me count, and oldest open age.
- Add assignment controls in the report detail panel.
- Add appeal detail and resolution controls in the report detail panel.
- Add enforcement history for the reported user with action, status, reviewer, date, and safe summary.
- Keep keyboard-accessible native controls and role=status/role=alert feedback.

## Recommendation

Use compact panels inside the existing admin work surface. Do not introduce new navigation or a dashboard homepage for this phase.
